Reusable MLOps Control Plane

A reusable orchestration layer that standardizes training, evaluation, packaging, smoke testing, policy decisions, registry updates, and model promotion across multiple machine learning projects.

Why this exists

This project simulates a real-world ML platform where models are trained, evaluated, validated through gates, and promoted or rejected based on measurable performance.

Architecture Overview

The control plane separates project-specific ML code from shared lifecycle orchestration. Each project implements an adapter, while the platform handles registry state, policy decisions, promotion logic, and CLI automation.

How to use the system

1. Build the Docker image

docker build -t mlops-control-plane .

2. Run the default CLI command

docker run --rm mlops-control-plane

3. Run tests

python -m pytest -q

4. Inspect system state

python -m mlops_orchestrator.cli status-all

5. Explain a decision

python -m mlops_orchestrator.cli explain gold

6. Trigger adaptive execution

python -m mlops_orchestrator.cli auto-run liar

Key Features

Adapter Pattern

Each ML project plugs into the same lifecycle contract without changing the orchestrator core.

Policy Engine

Promotion and rejection decisions are based on gates, metrics, smoke tests, and baseline comparison.

Model Registry

Candidates, promoted baselines, and decisions are tracked as structured JSON records.

Adaptive Runner

Lightweight automation layer that observes system state and decides whether a new run is required.

Why I built this

I built this control plane to move beyond isolated ML projects and practice the engineering layer around machine learning: lifecycle automation, promotion decisions, reproducibility, testing, and cross-project standardization.
