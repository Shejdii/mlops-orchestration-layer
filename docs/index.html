Key Features

Adapter Pattern

Each ML project plugs into the same lifecycle contract without changing the orchestrator core.

Policy Engine

Promotion and rejection decisions are based on gates, metrics, smoke tests, and baseline comparison.

Model Registry

Candidates, promoted baselines, and latest decisions are tracked as structured JSON records.

Adaptive Runner

A lightweight rule-based automation layer observes registry state and decides whether a run is needed.
